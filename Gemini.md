Gemini.md: アルゼンチンタンゴ・イントロクイズ開発プロジェクト
1. プロジェクト概要
アプリ名: アルゼンチンタンゴ イントロクイズ (tango-quiz)

目的: ミロンガで流れる曲名が覚えられない悩みを解決するため、イントロを聴いて「曲名・アーティスト・年代」を当てるクイズ形式の学習アプリ。

ターゲット: タンゴ愛好家、ダンサー（特に曲名を覚えたい人）。

2. 技術スタック・環境
フレームワーク: Next.js (App Router), React

スタイリング: Tailwind CSS

ホスティング: Vercel

ソース管理: GitHub (taaku-tech/tango-quiz)

開発ツール: VS Code, Claude Code (WSL/Ubuntu環境)

音源: Spotify Embed iframe (コンパクトタイプ: 80px)

3. 現在のアプリ仕様 (v1.4+)
問題構成: 全58曲（15楽団）からランダムに10問、または特定の演奏家別クイズ。

選択肢ロジック:
- 正解
- 同じ奏者の別曲
- 他の奏者の曲×2

UI/デザイン:
- カラー: ダークレッド（#8B0000）、ブラック、ゴールド（#C9A84C）。
- Spotifyプレイヤー: 曲名が見えないよう filter: blur(6px) を適用。
- ガイド: 金色の外枠（3px）と「右下をタップ」という操作案内。

音声読み上げ (TTS):
- 曲名・アーティスト: スペイン語 (es-ES)
- 日本語訳・年代: 日本語 (ja-JP)

追加コンテンツ:
- 演奏家紹介ページ（15名分の特徴・ダンスのポイント）。
- 楽団の系譜図ページ（lineage/page.js）。

4. データ構造 (songs.json)
現在、全58曲のタンゴの名曲データが格納されています。
```json
{
  "id": "1",
  "title": "La Cumparsita",
  "titleJa": "小さなカーニバルの行列",
  "artist": "Juan D'Arienzo y su Orquesta Típica",
  "year": 1943,
  "spotifyTrackId": "56m2q6ItHRx5HIF31YobrH",
  "description": "（歌詞の背景やダンスのポイントを詳しく記述）"
}
```

5. 技術的実装詳細
### ディレクトリ構成
- `app/`: ルーティングと各ページのメインロジック。
  - `page.js`: 総合クイズ（10問）。
  - `artists/page.js`: 15楽団の紹介。
  - `artist-quiz/page.js`: 特定演奏家選抜クイズ。
  - `lineage/page.js`: 楽団系譜のインタラクティブ・チャート。
- `components/`: UI部品群。
  - `QuizCard.js`: Spotifyプレイヤーの制御と選択肢表示。
  - `ChoiceButton.js`: TTS（音声読み上げ）機能を搭載した選択ボタン。
  - `SongModal.js`: 曲の背景解説を表示するモーダル。
- `data/`: `songs.json`（楽曲データベース）。

### 主要ロジック
- **クイズ生成 (`generateChoices`)**:
  - 正解曲と同じ奏者の曲を1つ、他奏者の曲を2つランダムに選出し、計4択を生成。
  - フィッシャー・イェーツのシャッフルを用いて問題と選択肢をランダム化。
- **Spotify埋め込み制御**:
  - `iframe`に`blur(6px)`を適用し、正解後または30秒経過後に解除（`transition: filter 0.4s`）。
  - 右端にパルスアニメーションのガイドを表示し、再生ボタンの場所を誘導。
- **音声読み上げ (TTS)**:
  - `SpeechSynthesisUtterance`を使用し、スペイン語(曲名/奏者)と日本語(邦題/年代)を交互に連続再生。
- **インタラクティブ系譜図**:
  - SVGを用いた座標マッピングにより、数百年にわたる楽団の関係性を可視化。
  - ドラッグ・ズーム機能、ノード選択による関係ハイライトを実装。

6. 開発時の重要ルール (必読)
ビルド確認: 修正後は必ず npm run build を実行し、ビルド成功を確認してから git push すること。

Spotifyの制約: iframe内部はCSSで操作不可。自動再生はブラウザポリシーで制限されるため、ユーザーの直接タップが必要。

30秒ルール: 未ログイン時は30秒で止まる仕様を前提とし、説明文でユーザーに周知する。

デザインの一貫性: フォントは Playfair Display（タイトル）を使用し、エレガントなタンゴの世界観を維持する。
